Code Style & Formatting

Indentation & Spacing

  • Use 2 spaces for indentation
  • No tabs, only spaces
  • Maximum line length: 100 characters
  • Use empty lines to separate logical blocks

Naming Conventions

  • Variables: camelCase (e.g., userName)
  • Functions: camelCase (e.g., getUserData())
  • Classes: PascalCase (e.g., UserService)
  • Constants: UPPER_SNAKE_CASE (e.g., MAX_RETRY_COUNT)

Comment Standards

  • Comment complex logic
  • Use JSDoc for functions
  • Avoid obvious comments
  • Keep comments up-to-date with code

Error Handling

  • Always handle exceptions
  • Use descriptive error messages
  • Log errors appropriately
  • Fail fast, fail loudly

Design Patterns

Singleton Pattern

Ensure a class has only one instance and provide global access to it.

Use cases: Database connections, logging, configuration managers

Factory Pattern

Create objects without specifying the exact class of object that will be created.

Use cases: Object creation with complex logic, plugin systems

Observer Pattern

Define a one-to-many dependency between objects so that when one changes state, all dependents are notified.

Use cases: Event systems, UI updates, notification systems

Repository Pattern

Mediate between the domain and data mapping layers using a collection-like interface for accessing domain objects.

Use cases: Data access layers, ORM implementations
